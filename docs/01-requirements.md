# Passly 요구사항 정의서

> 버전: 1.0 | 작성일: 2025-06

---

## 1. As-Is (현재 문제)

| 문제 | 설명 |
|------|------|
| 덤프 관리 불편 | PDF 파일을 직접 열어서 풀어야 함. 채점 불가. |
| 실전 연습 불가 | 타이머/셔플 없이 공부하면 실제 시험 환경과 괴리 |
| 오답 추적 없음 | 내가 무엇을 틀렸는지 누적 관리 불가 |
| 해설 접근 불편 | 해설 PDF를 별도로 열어서 대조해야 함 |
| AI 질문 불가 | 개념이 헷갈려도 문서를 직접 뒤져야 함 |

## 2. To-Be (Passly가 해결하는 것)

| 기능 | 해결 내용 |
|------|----------|
| PDF 업로드 | 덤프/가이드 PDF 업로드만 하면 자동으로 문제 추출 및 색인 |
| 실전 시험 | 실제 시험 시간 타이머 + 문제 셔플 + 일괄 채점 |
| AI 문제 생성 | 시험 가이드 기반으로 나올 가능성 높은 문제 자동 생성 |
| 오답 노트 | 틀린 문제만 모아서 재확인 및 해설 제공 |
| AI 질문 | 시험 가이드 기반 RAG 답변 + 출처 표시 |

## 3. 기능 요구사항

### /upload — 자료 업로드
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| U-01 | PDF 파일 드래그앤드롭 또는 클릭 업로드 | 필수 |
| U-02 | 자격증 선택 (업로드 전) | 필수 |
| U-03 | 자료 유형 선택: OFFICIAL_GUIDE / DUMP | 필수 |
| U-04 | 업로드 후 파이프라인 진행 상태 실시간 표시 (PENDING → PROCESSING → COMPLETED) | 필수 |
| U-05 | 업로드 이력 목록 및 상태 확인 | 필수 |
| U-06 | 파싱 실패 시 오류 메시지 표시 | 필수 |

### /exam — 실전 시험
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| E-01 | 시험 시작 전 설정: 문제 수, 영역 필터, 타이머 여부 | 필수 |
| E-02 | 1문제 1화면 레이아웃 | 필수 |
| E-03 | 실제 시험 시간 타이머 (Azure AI-102: 120분) | 필수 |
| E-04 | 문제 셔플 (매 시험마다 다른 순서) | 필수 |
| E-05 | 시험 종료 후 일괄 채점 | 필수 |
| E-06 | 채점 결과 화면: 정답률, 영역별 성과, 오답 목록 | 필수 |
| E-07 | 문제별 해설 표시 (정답 + 해설 텍스트 + 출처) | 필수 |
| E-08 | 덤프 문제 + AI 생성 문제 혼합 출제 선택 옵션 | 선택 |

### /practice — 연습 모드
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| P-01 | 시험 영역 선택 후 문제 수 지정 | 필수 |
| P-02 | AI가 해당 영역 나올 가능성 높은 문제 생성 | 필수 |
| P-03 | 문제 제출 즉시 채점 + 해설 표시 | 필수 |
| P-04 | 타이머 없음 | 필수 |

### /review — 오답 노트
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| R-01 | 내가 응시한 시험 세션 목록 | 필수 |
| R-02 | 세션별 오답 문제만 필터링 | 필수 |
| R-03 | 오답 문제 해설 다시 보기 | 필수 |
| R-04 | 특정 문제 북마크 기능 | 선택 |

### / — 대시보드
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| D-01 | 자격증별 전체 정답률 | 필수 |
| D-02 | 최근 5회 시험 결과 추이 | 필수 |
| D-03 | 영역별 오답률 Top 3 | 필수 |
| D-04 | 빠른 시작 버튼 (실전 시험 / 연습 모드) | 필수 |

### /chat — AI 질문 (선택)
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| C-01 | 자연어 질문 입력 | 선택 |
| C-02 | 시험 가이드 기반 RAG 답변 | 선택 |
| C-03 | 답변 출처 chunk 표시 | 선택 |

## 4. 데이터 요구사항

| 데이터 | 보존 정책 | 비고 |
|--------|---------|------|
| 업로드된 PDF | 영구 보존 | Cloud Storage |
| 문서 청크/임베딩 | 문서 비활성화 시까지 | pgvector |
| 시험 세션 | 영구 보존 | 이력 추적 |
| 풀이 이력(user_attempts) | 영구 보존, 삭제 금지 | 불변 이력 |

---

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 동시 접속 | 2~10명 동시 접속 지원 |
| 응답 시간 | 일반 API P95 < 1초, RAG 검색 P95 < 3초 |
| 비용 | 유저 없을 때 월 $20 이하 |
| 배포 | GCP Cloud Run (로컬 Docker Compose 우선 개발) |
| 보안 | API 키/비밀번호 Secret Manager 관리 |
| 확장성 | 자격증 추가 시 코드 변경 없이 데이터만 추가 |
